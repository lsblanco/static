Dialog ​

Componente
Beta

Los diálogs muestran contenido a través de una ventana que se superpone al resto del contenido de la página. Pueden contener información crucial o requerir una o varias acciones por parte del usuario.

Uso básico con v-model ​

La propiedad v-model permite enlazar el valor del componente con la vista.
Valor enlazado: false
Ver código
vue
<template>
  <ea-button type="text" @click="modelBasic = true">
      Abrir cuadro de diálogo
    </ea-button>
  <ea-dialog
      v-model="model"
  />
  <div class="m-t-8">Valor enlazado: {{ model }}</div>
</template>
ts
<script setup>
  import { ref } from 'vue';
  import { EaDialog, EaButton } from '@zurich-es-npm/ea-front-web-ui';

  const model = ref(false);
</script>
Con la propiedad visible podemos hacer que el componente sea visible. Por defecto el valor es falso.
Ver código
vue
<template>
  <ea-button type="text" @click="visible=true">
    Abrir cuadro de diálogo una vez
  </ea-button>
  <ea-dialog
    :visible="visible"
  ></ea-dialog>
</template>
ts
<script setup>
  import { ref } from 'vue';
  import { EaDialog, EaButton } from '@zurich-es-npm/ea-front-web-ui';

  const visible = ref(false);
</script>

Tipología ​

La propiedad type permite definir diferentes tipos de cuadro de diálogo, siendo los valores permitidos passive (por defecto), custom y fullscreen .

Adventencia

Los tipos transactional, danger y recognition, se mantienen por compatibilidad, pero recomendamos no utilizarlos porque en futuras versiones desaparecerán.

Pasivo (por defecto)
Personalizado
Pantalla completa
Ver código
vue
<template>
  <!---Pasivo (passive) por defecto-->
  <ea-button type="text" @click="modelVisiblePasivo = true">
    Abrir cuadro de diálogo
  </ea-button>
  <ea-dialog
    title="Título"
    v-model="modelVisiblePasivo"
  >
    <p>It is a long established fact that a reader will be distracted by the readable content of a page when looking at its layout. The point of using Lorem Ipsum is that it has a more-or-less normal distribution of letters, as opposed to using 'Content here, content here', making it look like readable English.</p>
  </ea-dialog>

  <!---Personalizado (Custom)-->
  <ea-button type="text" @click="modelVisibleCustom = true">
    Abrir cuadro de diálogo
  </ea-button>
  <ea-dialog
    title="Finalizar sesión"
    v-model="modelVisibleCustom"
    type="custom"
    :custom="{
      footer:{
        align: 'right',
        size: 'medium',
      },
      buttons:[
        {
          text: 'Cancelar',
          type: 'secondary',
        },
        {
          text: 'Aceptar',
          type: 'primary',
          class:'m-r-0',

      ]

    @button-click="modelVisibleCustom = false"
  >
    <p class="p-a-24">¿Desea cerrar su sesión para el resto de aplicaciones Zurich?</p>
  </ea-dialog>

  <!---Pantalla completa (fullscreen)-->
  <ea-button type="text" @click="modelVisibleFullscreen = true">
    Abrir cuadro de diálogo
  </ea-button>
  <ea-dialog
    title="Pantalla completa"
    v-model="modelVisibleFullscreen"
    type="fullscreen"
  >
  </ea-dialog>
<template>

Tamaños ​

La propiedad size permite definir cuatro tamaños del cuadro de diálogo, siendo los valores small, medium (por defecto), large, y extraLarge.
Small
Medium (por defecto)
Large
Extralarge
También podemos definir un tamaño fijo utilizando los valores fixed-small, fixed-medium, fixed-large, y fixed-extraLarge.
Fixed small (420px)
Fixed medium (570px)
Fixed large (720px)
Fixed extralarge (960px)
- La propiedad width permite definir el ancho del cuadro de dialogo en formato css.

NOTA IMPORTANTE

Se recomienda siempre utilizar la propiedad size anteriormente descrita en vez de la propiedad width.

Antes de cerrar ​

La propiedad before-close permite Devolución de llamada antes de que se cierre el cuadro de diálogo, y así evitarto si fuera necesario. La función recibe el parámetro done, que puede ser usado como método para cerrar el diálogo.
Ver código
vue
<template>
  <ea-button type="text" @click="modelVisibleClose = true">
    Abrir cuadro de diálogo
  </ea-button>
  <ea-dialog
    title="Título"
    content="Contenido"
    v-model="modelVisibleClose"
    :before-close="handleClose"
  />
<template>
ts
<script setup lang="ts">
  import { ref } from 'vue';
  import { EaDialog, EaButton } from '@zurich-es-npm/ea-front-web-ui';

  const modelVisibleClose = ref(false);
  const handleClose = (done) => {
    if (window.confirm('¿Seguro que quieres cerrar el cuadro de diálogo?')) {
      done();
    }
  };
</script>

Diálogos anidados ​

Normalmente no se recomienda los dialogos anidados, pero si necesita múltiples Diálogos mostrados en la página y anidar un diálogo dentro de otro, establezca la propiedad append-to-body del diálogo anidado como true, esto lo añadirá al body en lugar de su nodo padre, y así será renderizado correctamente.
Ver código
vue
<template>
  <ea-button type="text" @click="modelVisibleOuter= true">
      Abrir cuadro de diálogo
    </ea-button>
    <ea-dialog
      title="Cuadro de diálogo exterior"
      v-model="modelVisibleOuter"
    >
      <ea-dialog
        v-model="modelVisibleInner"
        width="500"
        title="Cuadro de diálogo interior"
        content="Esto es un cuadro de diálogo interior"
        append-to-body
      />
      Esto es un cuadro de diálogo exterior
      <template #footer>
        <ea-button class="m-r-16" type="secondary" @click="modelVisibleOuter = false">Cancel</ea-button>
        <ea-button  @click="modelVisibleInner = true">
          Abrir diálogo anidado
        </ea-button>
      </template>
    </ea-dialog>  
<template>

Contenido con slots ​

Se puede mostrar contenido personaliza en formato HTML en los diferentes slots que existen en el componente ea-dialog, aunque siempre que se pueda es recomendable usar la propiedad type, title y content, que garantizan el diseño y la usabilidad definidos por el sistema de diseño.
Contenido en el cuerpo
Se puede mostrar contenido en formato HTML en el cuerpo del diálogo entre las etiquetas del componente ea-dialog.
Contenido en la cabecera

Contenido en el pié

Contenido en título, cuerpo y pié
