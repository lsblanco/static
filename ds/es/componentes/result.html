Result ​

Componente
Beta

El componente Result se utiliza para proporcionar feedback al usuario sobre el resultado de una acción que ha realizado o para mostrar información importante del aplicativo. El objetivo del componente es comunicar de manera clara y concisa al usuario lo que ha sucedido en el aplicativo y proporcionarle la información necesaria para que pueda tomar decisiones informadas o seguir interactuando con el sistema de manera eficiente.

Tipología ​

La propiedad type permite definir el tipo del componente a utilizar, siendo los valores posibles response (por defecto), states y empty.
Response (por defecto)
Información
Si necesitas más información o ayuda, no dudes en contactarnos
States
Lo sentimos, ha sucedido algo inesperado
Ha ocurrido un error inesperado y no se puede completar la acción solicitada
Por favor, inténtelo de nuevo más tarde y si el problema persiste, póngase en contacto con nosotros
Empty
Sin datos
Ver código
vue
<template>
  <!-- Response (por defecto) -->
  <ea-result />
  
  <!-- States -->
  <ea-result type="states" />
  
  <!-- class="t-align-center">Empty -->
  <ea-result type="empty" />
</template>
ts
<script setup lang="ts">
  import { EaResult } from '@zurich-es-npm/ea-front-web-ui';
</script>

Casuíticas del tipo RESPONSE ​

La propiedad response-status permite definir el estado inicial del componente del tipo response, siendo sus valores posibles info (por defecto), success, warning y danger.
Info (por defecto)
Información
Si necesitas más información o ayuda, no dudes en contactarnos
Success
Operación realizada con éxito
Te informamos que la operación que realizaste se ha completado satisfactoriamente
Warning
Advertencia
Se ha presentado un problema que puede afectar el resultado de la operación
Danger
Lo sentimos, ha habido un error
No pudimos procesar tu solicitud en este momento
Ver código
vue
<template>
  <!-- Info (por defecto) -->
  <ea-result />
  
  <!-- Success -->
  <ea-result 
    response-status="success"
  /> 

  <!-- Warning -->
  <ea-result 
    response-status="warning"
  />
  
  <!-- Danger -->
  <ea-result 
    response-status="danger"
  />
</template>

<script setup lang="ts">
  import { EaResult } from '@zurich-es-npm/ea-front-web-ui';
</script>

Casuíticas del tipo STATES ​

La propiedad states-status permite definir el estado inicial del componente del tipo states, siendo los valores posibles basic (por defecto), not-found, session-app, session-expired, unauthorized, unavailable, maintenance, unexpected y time.
Basic (por defecto)
Lo sentimos, ha sucedido algo inesperado
Ha ocurrido un error inesperado y no se puede completar la acción solicitada
Por favor, inténtelo de nuevo más tarde y si el problema persiste, póngase en contacto con nosotros
Not found (states-status="not-found")
Página no encontrada
Lo sentimos, no pudimos encontrar la página que buscabas
Por favor, verifica si ingresaste la dirección correctamente o contáctanos si necesitas ayuda
Closed session of the current application (states-status="session-app")
Sesión cerrada
Tu sesión ha sido cerrada
Por favor, haz clic en el botón para volver a acceder a la aplicación
Session expired (states-status=session-expired)
Tu sesión ha expirado
Lo sentimos, tu sesión ha expirado debido a la inactividad o por seguridad
Por favor, haz clic en el botón para volver a acceder a la aplicación
Unauthorized (states-status=unauthorized)
No autorizado para acceder a esta página
Lo sentimos, no tienes permiso para acceder a esta página
Por favor, solicita acceso o contáctanos si necesitas ayuda
Page unavailable (states-status="unavailable")
Lo sentimos, esta aplicación no está disponible en este momento
Ahora mismo la aplicación no está disponible
Por favor, inténtalo de nuevo más tarde. ¡Gracias por tu paciencia y comprensión!
Page in maintenance (states-status="maintenance")
Estamos haciendo mejoras
Lo sentimos, estamos realizando un mantenimiento en la página para mejorar tu experiencia
La página volverá a estar disponible pronto. Por favor, disculpa las molestias
Something unexpected has happened (states-status=unexpected)
Lo sentimos, ha sucedido algo inesperado
Ha ocurrido un error inesperado y no se puede completar la acción solicitada
Por favor, inténtelo de nuevo más tarde y si el problema persiste, póngase en contacto con nosotros
Waiting time exceeded (states-status=time)
Tiempo de espera excedido
Lo sentimos, el tiempo de espera para completar esta acción ha excedido su límite
Por favor, haz clic en el botón para actualizar la página
