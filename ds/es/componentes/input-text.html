Input Text ​

Componente
Beta

El componente InputText permite al usuario introducir respuestas de texto libre.

Uso básico ​

Con la propiedad value podemos establecer el valor del componente.
Ver código
vue
<template>
  <ea-input-text 
    value="texto" 
  />
</template>
ts
<script setup>
  import { EaInputText } from '@zurich-es-npm/ea-front-web-ui';
</script>

Uso básico con v-model ​

La propiedad v-model permite enlazar el valor del componente con la vista.
Valor enlazado: hola
Ver código
vue
<template>
  <ea-input-text 
    v-model="model"
  />
  <div class="m-t-8">Valor enlazado: {{ model }}</div>
</template>
ts
<script setup>
  import { EaInputText } from '@zurich-es-npm/ea-front-web-ui';

  const model = ref('');
</script>

Placeholder ​

Con la propiedad placeholder podemos añadir un texto cuando el componente no tiene todavía un valor.

Con placeholder

vue

    <!-- Con placeholder -->
    <ea-input-text 
      placeholder="Introduce un valor..."

Clearable ​

Con la propiedad clearable podemos activar la funcionalidad de limpiado del contenido del input mediante un icono que se monstrará cuando nos posicionamos en el input y existe algún carácter.
Ver código
vue
<template>
  <ea-input-text
    placeholder="Introduce un valor..."
    clearable
    @clear="handleClear"
  />
</template>
ts
<script setup lang="ts">
  import { EaInputText } from '@zurich-es-npm/ea-front-web-ui';

  const handleClear = () => {
    console.log(`el texto se ha borrado.`);
  };
</script>

Iconos ​

Con las propiedades prefixicon y suffixicon podemos añadir un icono para indicar la tipología del campo.
prefixicon (icono a la izquierda)
suffixicon (icono a la derecha)

vue
<template>
  <!-- prefixicon (icono a la izquierda) -->
  <ea-input-text
    placeholder="Introduce un valor..."
    prefix-icon="z-search">
  </ea-input-text>

  <!-- suffixicon (icono a la derecha) -->
  <ea-input-text
    placeholder="Introduce un valor..."
    suffix-icon="z-search">
  </ea-input-text>
</template>
ts
<script setup lang="ts">
  import { EaInputText } from '@zurich-es-npm/ea-front-web-ui';
</script>

Límite de carácteres ​

La propiedad showwordlimit junto con la propiedad maxlength, muestra el contador de caracteres con el límite de la cantidad de caracteres que un usuario puede introducir.
0 / 30
Ver código
vue
<template>
  <ea-input-text
     placeholder="Máximo 30 caracteres"
    showwordlimit
    :maxlength="30"
  ></ea-input-text>
</template>
ts
<script setup lang="ts">
  import { EaInputText } from '@zurich-es-npm/ea-front-web-ui';
</script>

Tipo de teclado en dispositivos móviles ​

La propiedad inputmode permite seleccionar el tipo de teclado en versiones para dispositivos móviles, siendo los valores permitidos text (por defecto), decimal, numeric, tel, search, email, url, y none.
Tipo de teclado
Text (por defecto)
Decimal
Numeric
Tel
Search
Email
Url
None
Ver código
vue
<template>
  <!-- Text (por defecto) -->       
  <ea-input-text/>
  <!-- Decimal -->      
  <ea-input-text 
    inputmode="decimal"
  /> 
  <!-- Numeric -->      
  <ea-input-text
    inputmode="numeric"
  />
  <!-- Tel -->       
  <ea-input-text 
    inputmode="tel"
  />   

  <!-- Search -->     
  <ea-input-text
    inputmode="search" 
  />
  <!-- Email -->       
  <ea-input-text
    inputmode="email" 
  />
  <!-- Url -->      
  <ea-input-text
    inputmode="url" 
  />
  <!-- None -->      
  <ea-input-text 
    inputmode="none" 
  />  
</template>
ts
<script setup lang="ts">
  import { EaInputText } from '@zurich-es-npm/ea-front-web-ui'
</script>
Se recomienda utilizar el tamaño large o medium en el input ya que automáticamente al detectar que es un dispositivo móvil, el tamaño de la tipografía en el input pasa de 14px a ser de 16px. Este tamaño permite que no se aplique el zoom en el input, cuyo efecto no es el deseado.
Tamaños medium y large NO aplican zoom (es lo óptimo en móvil)
Tamaños mini y small aplican zoom (no utilizar en móvil)
Ver código
vue
<template>
  <!-- Tamaños medium y large NO aplican zoom (es lo óptimo en móvil) -->       
  <ea-input-text size="large"> </ea-input-text>

  <!-- Tamaños mini y small aplican zoom (no utilizar en móvil) -->        
  <ea-input-text size="small"> </ea-input-text>
</template>
ts
<script setup lang="ts">
  import { EaInputText } from '@zurich-es-npm/ea-front-web-ui';
</script>

Con la propiedad customVar podemos personalizar el diseño del componente mediante las siguientes variables:

--ea-input-text-focus-border-color--color-primaryColor del borde al realizar focus
Color del borde
Color del borde en el hover
Color del borde en el focus
Colores en el borde, hover y focus

Casos de uso ​

Caso de uso de funcionamiento de eventos ​

El componente input dispone de los eventos focus, input, change, y blur, que se disparan en el orden anteriormente mencionado.
Ver código
vue
<template>
  <ea-input-text 
    @focus="handleFocus"
    @input="handleInput"
    @change="handleChange"
    @blur="handleBlur"
  />
</template>
ts
<scrip setup>
  import { EaInputText } from '@zurich-es-npm/ea-front-web-ui';

  const handleFocus = (event) => {
    console.log(`Evento FOCUS con valor ${event.type}`);
  };

  const handleInput = (value) => {
    console.log(`Evento INPUT con valor "${value}"`);
  };

  const handleChange = (value) => {
    console.log(`Evento CHANGE con valor ${value}`);
  };

  const handleBlur = (event) => {
    console.log(`Evento BLUR con valor ${event.type}`);
  };
</script>
También disponemos del evento clear que se dispara cuando pulsamos el icono de Borrar.
Ver código
vue
<template>
  <ea-input-text
    clearable
    @clear="handleClear"
  />
</template>
ts
<script setup lang="ts">
  import { EaInputText } from '@zurich-es-npm/ea-front-web-ui';

  const handleClear = () => {
    console.log(`Evento CLEAR`);
  };
</script>

Caso de uso de requeridos en un formulario Pediente
​

El componente ea-input-text pueder ser utilizado como un campo de formulario sobre el cual se pueden aplicar ciertas validaciones. Para indicar que el campo de texto es obligatorio se ha de hacer a nivel de ea-form-item fijando la propiedad required a true.
Para obtener más información acerca de validaciones en los campos del formulario se pueden consultar la guías de desarollo
Ver código
vue
<template>
  <ea-form :model="ruleForm">
    <ea-form-item
      label="Nombre"
      prop="name"
      class="m-b-24"
      required
    >
      <ea-input-text
        placeholder="Escribe algo" v-model="ruleForm.name"
      ></ea-input-text>
    </ea-form-item>
  </ea-form>
</template>
ts
<script setup lang="ts">
  import { EaInputText } from '@zurich-es-npm/ea-front-web-ui';

  export default {
    data () {
      return {
        ruleForm: {
          name: ''
        }
      };
    }
  }
</script>
