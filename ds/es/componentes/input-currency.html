Input Currency ​

Componente
Terminado

Permite al usuario introducir respuestas de texto moneda.

Uso básico ​

Con la propiedad value podemos establecer el valor del input.
Ver código
vue
<template>
  <ea-input-currency 
    :value="12" 
  />
</template>
ts
<script setup>
  import { EaInputCurrency } from '@zurich-es-npm/ea-front-web-ui';
</script>

Uso básico con v-model ​

Con la propiedad v-model podemos establecer el enlace a datos.
value: 10
Ver código
vue
<template>
  <ea-input-currency 
    v-model="modelBasic" 
  />
  <code>value: {{ modelBasic }}</code>
</template>
ts
<script setup>
  import { EaInputCurrency } from '@zurich-es-npm/ea-front-web-ui';

  const modelBasic = ref(10);
</script>

Placeholder ​

Con la propiedad placeholder podemos añadir un texto cuando el componente no tiene todavía un valor.
Sin placeholder
Con placeholder
Ver código
vue
<template>
  <!-- Sin placeholder -->
  <ea-input-currency />

  <!-- Con placeholder -->
  <ea-input-currency 
    placeholder="Introduce un valor..." 
  />
</template>
ts
<script setup>
  import { EaInputCurrency } from '@zurich-es-npm/ea-front-web-ui';
</script>

Tamaños ​

Con la propiedad size podemos indicar el tamaño en altura, siendo los valores posibles mini, small (default), medium y large.
Mini
Small (default)
Medium
Large
Ver código
vue
<template>
  <!-- Mini -->
  <ea-input-currency 
    size="mini" 
    placeholder="mini (28px)" 
  />

  <!-- Small -->
  <ea-input-currency 
    size="small" 
    placeholder="small (32px)" 
  />

  <!-- Medium -->
  <ea-input-currency 
    size="medium" 
    placeholder="medium (36px)" 
  />

  <!-- Large -->
  <ea-input-currency 
    size="large" 
    placeholder="large (40px)" 
  />
</template>
ts
<script setup lang="ts">
  import { EaInputText } from '@zurich-es-npm/ea-front-web-ui'
</script>

Deshabilitado y solo lectura ​

Con las propiedades disabled y readonly, el componente se mostrará deshabilitado o de solo de lectura.
Deshabilitado
Solo lectura
Ver código
vue
<template>
  <doc-sub>Deshabilitado</doc-sub>
  <ea-input-currency 
    :value="20" 
    disabled 
  />

  <doc-sub>Solo lectura</doc-sub>
  <ea-input-currency 
    :value="20" 
    readonly 
  /> 
</template>
ts
<script setup>
  import { EaInputCurrency } from '@zurich-es-npm/ea-front-web-ui';
</script>

Habilitar/Deshabilitar valores negativos ​

A través de la propiedad allowNegative podemos habilitar/deshabilitar valores numércios negativos. Por defecto el valor de esta prop es true.
Habilitado (por defecto)
Deshabilitado
Ver código
vue
<template>
  <!-- Habilitado (por defecto) -->
  <ea-input-currency
    :value="-15"
  />

  <!-- Deshabilitado -->
  <ea-input-currency
    :value="-15"
    :allowNegative="false"
  /> 
</template>
ts
<script setup>
  import { EaInputCurrency } from '@zurich-es-npm/ea-front-web-ui';
</script>

Habilitar/Deshabilitar separador de números ​

Con la propiedad booleana useGrouping podemos habilitar o deshabilitar el separador de números, siendo los valores posibles true (valor inicial) o false.
Habilitado (de inicio)
value: 1000
Deshabilitado
value: 1000
Ver código
vue
<template>
  // Habilitado (de inicio)
  <ea-input-currency
    v-model="modelUseGroupingTrue"
  />
  <code>value: {{ modelUseGroupingTrue }}</code>
      
  // Deshabilitado
  <ea-input-currency
    v-model="modelUseGroupingFalse"
    :useGrouping="false"
  />
  <code>value: {{ modelUseGroupingFalse }}</code>

</template>
ts
<script setup>
  import { EaInputCurrency } from '@zurich-es-npm/ea-front-web-ui';

  const modelUseGroupingTrue = ref(1000);
  const modelUseGroupingFalse = ref(1000);
</script>

Límite de carácteres ​

Con la propiedad númerica maxlength, define un límite en la cantidad de caracteres que un usuario puede introducir.
Máximo 5 caracteres
Máximo 15 caracteres
Ver código
vue
<template>
  <!-- Máximo 5 caracteres -->
  <ea-input-currency
    v-model="value1"
    :maxlength="5"
  />

  <!-- Máximo 15 caracteres -->
  <ea-input-currency
    v-model="value2"
    :maxlength="15"
  />
</template>
ts
<script setup>
  import { EaInputCurrency } from '@zurich-es-npm/ea-front-web-ui';

  const value1 = ref(12345);
  const value2 = ref(123456789012345);
</script>

Precisión ​

Con la propiedad precision podemos configurar el número de dígitos decimales que se podrán introducir por teclado y el resto se complementaran con ceros. La precisión puede estar entre 0 y 16.
Precisión por defecto (0)
value: 30
precision = 3
value: 30.111
precision = 16
value: 0.1234567890123456
Ver código
vue
<template>
  <!-- Precisión por defecto (0) -->
  <ea-input-currency v-model="valuePrecision1" />
  <div class="m-t-8">value: {{ modelPrecision1 }}</div>

  <!-- precision = 3 -->
  <ea-input-currency
    v-model="modelPrecision2" 
    :precision="3" 
  />
  <div class="m-t-8">value: {{ modelPrecision2 }}</div>

  <!-- precision = 16 -->
  <ea-input-currency v-model="modelPrecision3" :precision="16" />
  <div class="m-t-8">value: {{ modelPrecision3 }}</div>
</template>
ts
<script setup>
  import { EaInputCurrency } from '@zurich-es-npm/ea-front-web-ui';

  const modelPrecision1 = ref(0.11);
  const modelPrecision2 = ref(0.111);
</script>

Advertencia

El numero de la precisión puede estar entre 0 y 16, pero ten en cuenta que la suma de los dígitos de la parte entera y la suma de los dígitos de la parte décimal no puede ser superior a 17.

También podemos incluir en la propiedad precisión un objeto {min, max} para usar rango de precisión.
precision = { min: 2, max: 4 }
value: 30
Ver código
vue
<template>
  <doc-sub>precision = { min: 2, max: 4 }</doc-sub>
  <ea-input-currency v-model="modelPrecisionMinMax"
    :precision="{
      min: 2,
      max: 4
    }"
  />
  <div class="m-t-8">value: {{ modelPrecisionMinMax }}</div>
ts
<script setup>
  import { EaInputNumber } from '@zurich-es-npm/ea-front-web-ui';

  const modelPrecisionMinMax = ref(0.1234567890123456)
</script>

Importante

Para que el rango de validaciones funcione correctamente es necesario las dos propiedades min y max . El numero de la precisión en ambas propiedades puede estar entre 0 y 16, pero el valor de min no puede ser superior al valor de max y viceversa.

Ten en cuenta que la suma de dígitos de la parte entera y de la parte décimal no puede ser superior a 17.

Prefijo y sufijo ​

Con la propiedad prefix y suffix podemos personalizar el símbolo inicial y final del número de moneda.
Prefix
value: 1000
Suffix
value: 1000
Prefix/Suffix
value: 1000
Ver código
vue
<template>
   <!-- Prefix -->
    <ea-input-currency
      v-model="modelPrefix" 
      prefix="#" 
    />
    <div class="m-t-8">value: {{ modelPrefix }}</div>
 
    <!-- Suffix -->
    <ea-input-currency 
      v-model="modelSuffix" 
      suffix="%" 
    />
    <div class="m-t-8">value: {{ modelSuffix }}</div>
  
    <!-- Prefix/Suffix -->
    <ea-input-currency 
      v-model="modelPrefixSuffix" 
      prefix="#" 
      suffix="%" 
    />
    <div class="m-t-8">value: {{ modelPrefixSuffix }}</div>
</template>
ts
<script setup>
  import  { ref } from vue;
  import { EaInputCurrency } from '@zurich-es-npm/ea-front-web-ui';

  const modelPrefix = ref(1000);
  const modelSuffix = ref(1000);
  const modelPrefixSuffix = ref(1000);
</script>

Configuración del Currency/Locale ​

Con la propiedad currency y locale podemos indicar el código de la moneda a utilizar (EUR, USD y GBP) y el código del idioma local a aplicar (es-ES, en-US y en-GB), siendo los valores iniciales EUR , y es-ES.
Default
value: 1000
USD/en-US
value: 1000
GBP/en-GB
value: 1000
Ver código
vue
<template>
  <!-- Default -->
  <ea-input-currency
    v-model="modelConfig"
  />
  <div class="m-t-8">value: {{ modelConfig }}</div>
  
  <!-- USD/en-US -->
  <ea-input-currency
    v-model="valueConfigUsa"
    currency="USD"
    locale="en-US"
  />
  <div class="m-t-8">value: {{ modelConfigUsa }}</div>

  <!-- GBP/en-IN -->
  <ea-input-currency
    v-model="modelConfigEn"
    currency="GBP"
    locale="en-IN"
  />
  <div class="m-t-8">value: {{ modelConfigEn }}</div>
</template>
ts
<script setup>
  import { EaInputCurrency } from '@zurich-es-npm/ea-front-web-ui';

  const modelConfig = ref(1000);
  const modelConfigUsa = ref(1000);
  const modelConfigEn = ref(1000);
</script>
