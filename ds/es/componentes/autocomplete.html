Autocomplete ​

Componente
Beta

El componente autocomplete detecta y rellena las entradas cuando el usuario edita los campos para reducir los errores y ahorrar tiempo.

Uso básico ​

Uso básico del componente ea-autocomplete. Para generar las sugerencias del componente se ha capturar el evento fetchSuggestions e implementar toda la lógica (en el caso de este ejemplo se implementa dentro del método querySearch). Una vez obtenemos los datos que se quieren sugerir se han de devolver al componente Autocomplete a través de la función callback proporcionada. En el caso de usar una lista de objetos como sugerencias, es necesario el uso de la prop displayProperty para indicar qué campo se tiene que mostrar en el componente.
Ver código
vue
<template>
  <ea-autocomplete
    placeholder="Introduce..."
    displayProperty="value"
    @suggestionSelected="onSuggestionSelectedAction"
    @fetchSuggestions="querySearch"
  />
</template>
ts
<script setup lang="ts">
  import { EaAutocomplete } from '@zurich-es-npm/ea-front-web-ui';
  import { ref } from 'vue';

  const repositories = ref([
    {'value': 'vue', 'link': 'https://github.com/vuejs/vue'},
    {'value': 'element', 'link': 'https://github.com/ElemeFE/element'},
    {'value': 'cooking', 'link': 'https://github.com/ElemeFE/cooking'},
    {'value': 'mint-ui', 'link': 'https://github.com/ElemeFE/mint-ui'},
    {'value': 'vuex', 'link': 'https://github.com/vuejs/vuex'},
    {'value': 'vue-router', 'link': 'https://github.com/vuejs/vue-router'},
    {'value': 'babel', 'link': 'https://github.com/babel/babel'}
  ]);

  const createFilter = (queryString) => {
    const callback = (repository) => {
      return repository.value.toLowerCase().indexOf(queryString.toLowerCase()) === 0;
    }
    return callback;
  }

  const querySearch = (fetchSuggestionsData) => {
    const results = fetchSuggestionsData.query ?
      repositories.value.filter(createFilter(fetchSuggestionsData.query)): repositories.value;
    fetchSuggestionsData.callback(results);
  }

  const onSuggestionSelectedAction = (value) => {
    console.log(`Evento SuggestionSelected con valor "${JSON.stringify(value)}"`);
  }
</script>

Uso básico con value ​

Con la propiedad value podemos establecer el valor del componente.
Ver código
vue
<template>
  <ea-autocomplete

valor enlazado:
{ "value": "vue", "link": "https://github.com/vuejs/vue" }

vue
<template>

    v-model:value="modelBasic"
    placeholder="Introduce..."
    displayProperty="value"

    @suggestionSelected="onSuggestionSelectedAction"
  />
  <doc-sub>
    valor enlazado:
    <pre v-if="modelBasic">{{modelBasic}}</pre>
  </doc-sub>
</template>

Tamaños ​

Con la propidedad size podemos indicar al componente ea-autocomplete el tamaño de la altura, siendo los valores posibles mini, small (por defecto), medium y large.
mini (28px)
small (32px)
medium (36px)
large (40px)

Ancho ​

Con la propiedad width definimos el ancho del componente, siendo el valor por defecto auto. Esta propiedad nos permite indicar un valor con las unidades de medida en pixel, em, vw y porcentaje, pero si no ponemos ninguna unidad css con el tipo string o number, cogerá pixels.
Ancho con valor auto (Por defecto)
Ancho con valor 200 (sin unidades CSS)
Ancho con valor 200px
Valor del 80% del ancho del contenedor en el que está incluido
Valor del 100% del ancho, para que se adapte al ancho de su contenedor
Ver código
vue
<template>
  <!-- Ancho con valor auto (Por defecto) -->
  <ea-autocomplete />
   
  <!-- Ancho con valor 200 (sin unidades CSS) -->
  <ea-autocomplete width="200" />
    
  <!-- Ancho con valor 200px -->
  <ea-autocomplete width="200px" />
    
  <!-- Valor del 80% del ancho del contenedor en el que está incluido -->
  <ea-autocomplete width="80%" />
   
  <!-- Valor del 100% del ancho, para que se adapte al ancho de su contenedor -->
  <ea-autocomplete width="100%" />
</template>
ts
<script setup lang="ts">
  import { EaAutocomplete } from '@zurich-es-npm/ea-front-web-ui';
</script>

Estados ​

Dependiendo del estado del componente, éste se tendrá que mostrar con unos estilos determinados. A continuación se muestran los diferentes estados que podría adoptar el componente ea-autocomplete.
Default
Disabled
Read only
Filled
Ver código
vue
<template>
  <!-- Default -->
  <ea-autocomplete
    displayProperty="value"
    @fetchSuggestions="querySearch"
    placeholder="Introduce..."
    value="Zurich"
  />
 
  <!-- Disabled -->
  <ea-autocomplete
    displayProperty="value"
    @fetchSuggestions="querySearch"
    placeholder="Introduce..."
    disabled
    value="Zurich"
  />
  
  <!-- Read only -->
  <ea-autocomplete
    displayProperty="value"
    @fetchSuggestions="querySearch"
    placeholder="Introduce..."
    :value="autoCompleteDefault"
    readonly
    value="Zurich"
  /> 
  
  <!-- Filled -->
  <ea-autocomplete
    displayProperty="value"
    @fetchSuggestions="querySearch"
    placeholder="Introduce..."
    :value="autoCompleteDefault"
    value="Zurich"
  />
</template>
ts
<script setup lang="ts">
  import { EaAutocomplete } from '@zurich-es-npm/ea-front-web-ui';

Muestra de la lista de sugerencias en el foco ​

Mediante la propiedad booleana trigger-on-focus se muestran las sugerencias de la lista al obtener en el foco del componente, siendo los valores posibles false (por defecto), dónde no se muestran y true, mostrándose.
No se muestran las sugerencias (por defecto)
Se muestran las sugerencias al obtener el foco
Ver código
vue
<template>
  <!-- No se muestran las sugerencias al obtener el foco (por defecto) -->
  <ea-autocomplete
    placeholder="Introduce..."
    displayProperty="value"
    @fetchSuggestions="querySearch"
    @suggestionSelected="onSuggestionSelectedAction"
  />

  <!-- Se muestran las sugerencias al obtener el foco -->
  <ea-autocomplete
    placeholder="Introduce..."
    displayProperty="value"
    @fetchSuggestions="querySearch"
    @suggestionSelected="onSuggestionSelectedAction"
    :trigger-on-focus="true"
  />
</template>
ts
<script setup lang="ts">
  import { EaComplete } from '@zurich-es-npm/ea-front-web-ui'
</script>

Casos de uso ​

Caso de uso de funcionamiento de eventos ​

El componente Autocomplete dispone de los eventos focus, input, change, y blur, que se disparan en el orden anteriormente mencionado. También disponemos del evento clear que se dispara cuando pulsamos el icono de Borrar.
Ver código
vue
<template>
  <ea-autocomplete
    @fetchSuggestions="querySearch"
    @focus="handleFocus"
    @input="handleInput"
    @change="handleChange"
    @blur="handleBlur"
  ></ea-autocomplete> 
</template>
ts
<script setup lang="ts">
  import { EaAutocomplete } from '@zurich-es-npm/ea-front-web-ui';
  import { ref } from 'vue';

  const repositories = ref([
    {'value': 'vue', 'link': 'https://github.com/vuejs/vue'},
    {'value': 'element', 'link': 'https://github.com/ElemeFE/element'},
    {'value': 'cooking', 'link': 'https://github.com/ElemeFE/cooking'},
    {'value': 'mint-ui', 'link': 'https://github.com/ElemeFE/mint-ui'},
    {'value': 'vuex', 'link': 'https://github.com/vuejs/vuex'},
    {'value': 'vue-router', 'link': 'https://github.com/vuejs/vue-router'},
    {'value': 'babel', 'link': 'https://github.com/babel/babel'}
  ]);

  const querySearch = (fetchSuggestionsData) => {
    setTimeout(() => {
      const results = fetchSuggestionsData.query ?
        repositories.value.filter(createFilter(fetchSuggestionsData.query)): repositories.value;
      fetchSuggestionsData.callback(results);
    }, 500);
  }

  const createFilter = (queryString) => {
    const callback = (repository) => {
      return repository.value.toLowerCase().indexOf(queryString.toLowerCase()) === 0;
    }
    return callback;
  }

  const handleFocus = (event) => {
    if (event) {
      console.log(`Evento focus con valor "${event.type}"`);
    }
  }

  const handleInput = (value) => {
    console.log(`Evento input con valor "${JSON.stringify(value)}"`);
  }

  const handleChange = (value) => {
    console.log(`Evento Change con valor "${value}"`);
  }

  const handleBlur = (event) => {
    if (event) {
      console.log(`Evento Blur con valor "${event.type}"`);
    }
  }

  const handleClear = () => {
    console.log(`Evento Clear`);
  }
</script>
