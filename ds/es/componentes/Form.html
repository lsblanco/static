Formulario con campos obligatorios Terminado ​

Con la propiedad booleana required en el componente ea-form-item dentro de un componente formulario ea-form, obligamos a que el campo sea obligatorio. Esta acción mostrará un asterisco de color rojo y un mensaje de 'Campo requerido' cuando se pierda el foco (evento BLUR), y no haya ningún cáracter en el campo.
Campo obligatorio
Campo no obligatorio
Ver código
vue
<template>
  <ea-form :model="modelRequired" class="m-t-16" >
  <ea-row>
    <ea-col :sm="24" :md="12" :lg="12" :xl="12">
      // Campo obligatorio
      <ea-form-item
        label="Nombre"
        prop="nombre"
        required
      >
        <ea-input-text v-model="modelRequired.nombre"/>
      </ea-form-item>
    </ea-col>
    <ea-col :sm="24" :md="12" :lg="12" :xl="12">
      // Campo no obligatorio
      <ea-form-item
        label="Apellidos"
        prop="apellidos"
      >
        <ea-input-text v-model="modelRequired.apellidos"/>
      </ea-form-item>
    </ea-col>
  </ea-row>
</ea-form>
</template>

Formulario con reglas de validación Falta Ver código ​

Exiten reglas de validación de rango, de longitud, de patrón, de enumerados, personalizadas y de tipo. En estas reglas de validación opcionalmente se puede personalizar el mensaje, el evento donde se desencadenarán, y el tipo de dato a validar. Por defecto la reglas de validación tienen un mensaje estándar traducido al idioma selecionado, se desencadenan en el evento blur y están asignadas al tipo string.

IMPORTANTE

Para que las reglas de validación funcionen correctamente, debemos asignar correctamente el tipo de dato a validar que tiene que coincidir con el tipo de dato asignado y permitido en el vmodel/value del componente. Veamos unos ejemplos:

1) Si tenemos dentro del form-item un componente ea-input-number cuya propiedad value/vmodel es un number, debemos indicar el tipo EaValidationTypes.NUMBER: eaRangeValidation(5, 20, 'mensaje', EaValidationTriggers.BLUR, EaValidationTypes.NUMBER)
2) Si tenemos dentro del form-item un componente ea-date-picker cuya value/vmodel tiene como posibles tipos Date | string | Date[] | string[], y asignamos como valor un objeto Date, debemos indicar el tipo EaValidationTypes.DATE: eaLengthValidation(8, 'mensaje', EaValidationTriggers.BLUR, EaValidationTypes.DATE)
Regla de validación de rango
Valida si la longitud de un campo está entre el valor mínimo y máximo definido en la regla.
- eaRangeValidation(5, 20)
- eaRangeValidation(5, 20, 'los apellidos tienen que tener entre 5 y 20 carácteres.', EaValidationTriggers.BLUR, EaValidationTypes.STRING)
- eaRangeValidation(1, 99, 'La edad tiene que tener un valor entre 1 y 99.', EaValidationTriggers.BLUR, EaValidationTypes.NUMBER)
Regla de validación de longitud
Valida en un campo el número de carácteres permitidos.
- eaLengthValidation(10,'El número de caráteres permitidos es 10.', EaValidationTriggers.BLUR, EaValidationTypes.STRING)
Regla de validación de patrón
Valida si el valor del campo cumple con la expresión regular indicada.
- eaPatternValidation(/^\d{8}[A-Z]$/)
Regla de validación de enumerados
Valida si el valor del campo es uno de los definidos en la regla.
- eaEnumValidation(['A', 'B', 'C'])
Regla de validación de tipos
Valida si el valor del campo se corresponde con los tipos email, url, string, number, integer, float, boolean, date, array, object, method, y hexadecial, definidos. Nota importante: Para que la regla de valicadicón funcione correctamente, es necesario que el vmodel/value del componente a utilizar tenga el mismo tipo que la regla definida.
- eaTypeValidation(EaValidationTypes.EMAIL, 'Email no válido.')
- eaTypeValidation(EaValidationTypes.URL, 'Url no válida')
Regla de validación personalizada
Valida una regla definida mediante una función para un campo determinado.
- eaCustomValidation((rule, value, callback) => { ... }, 'La letra del dni no es válida.')
